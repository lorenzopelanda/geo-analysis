get_green(**kwargs) → dict[source]#

Filters and processes green areas from the Copernicus data.

Parameters:

**kwargs (dict, optional) –

Additional arguments to specify green areas. - green_areas (frozenset, optional): A set of values representing green areas

(default is frozenset([10, 20, 30])).

Returns:

A dictionary containing the filtered green areas with the following keys:

  • ’data’: The binary raster data where green areas are marked as 1.

  • ’transform’: The affine transform of the raster.

  • ’crs’: The coordinate reference system of the raster.

  • ’shape’: The shape of the raster.

Return type:

dict

Notes

The method uses the numpy library to filter raster data and the tqdm library to display progress.

get_green(**kwargs) → Tuple[GeoDataFrame, GeoDataFrame][source]#

Filters and processes green areas from the OSM data.

Parameters:

**kwargs (dict, optional) –

Additional arguments to specify green tags. - green_tags (dict, optional): A dictionary of tags to filter green areas.

Default is: {

’natural’: [‘wood’, ‘tree_row’, ‘tree’, ‘scrub’, ‘grassland’,

’heath’, ‘fell’, ‘tundra’, ‘shrubbery’],

’landuse’: [‘forest’, ‘meadow’, ‘grass’, ‘allotments’], ‘leisure’: [‘park’, ‘garden’, ‘nature_reserve’]

}

Returns:

A tuple containing two GeoDataFrames: - green_nodes (geopandas.GeoDataFrame): Filtered green nodes. - green_edges (geopandas.GeoDataFrame): Filtered green edges.

Return type:

tuple

Raises:

Exception – If an error occurs during processing, returns empty GeoDataFrames.

Notes

The method uses the pandas and geopandas libraries to filter OSM data based on specified tags.
